## NAT 限制

内网设备都在校园网 NAT 后面：

```text
5090:      172.16.14.240
新设备:     172.16.14.17
出口 NAT:   123.127.159.1
```

8060S 当前无法使用，暂不纳入网络拓扑和端口规划。

新设备的 RTX 5080 16GB + RTX 4060 Ti 16GB 可以按资源池理解为 32GB 专用显存，但不是单卡连续显存。Windows 显示的共享 GPU 内存不应按 VRAM 规划；网络和路由规划应把它当作两张独立 GPU 看待。

限制：

- 外部无法直接访问内网设备。
- 没有公网 IP 和端口映射权限。
- 实验室限制下不优先使用 FRP / Tailscale / ZeroTier。

解决方案：每台需要对外提供服务的设备主动建立 SSH Reverse Tunnel 到云服务器。当前 5090 需要手动开启 `:12340` 隧道，新设备需要手动开启 `:12341` 隧道。RAG Service v1 运行在 5090 本地 `127.0.0.1:8010`，需要公网远程验证时从 5090 额外开启 `0.0.0.0:18010 -> 127.0.0.1:8010` 反向隧道；云端 sshd 已设置 `GatewayPorts clientspecified`，腾讯云安全组已放行 TCP 18010。

## LM Studio 监听策略

推荐生产/长期使用时让每台本地 LM Studio 只监听 `127.0.0.1:1234`，再由本机 SSH 反向隧道转发到云服务器。这样内网其他机器不能绕过 LiteLLM 直接调用模型服务。

```text
127.0.0.1:1234       仅本机可访问，适合正式隧道路由
172.16.14.x:1234     局域网可访问，适合临时调试
82.156.69.153:8000   公网统一入口，带 LiteLLM API Key
```

如果临时开启 LM Studio 的 `Serve on Local Network`，建议只在调试阶段使用，并用 Windows 防火墙限制来源 IP。

## SSH Reverse Tunnel

命令格式：

```bash
ssh -N -R <云服务器端口>:127.0.0.1:<本机端口> ubuntu@82.156.69.153
```

建议参数：

```text
ServerAliveInterval=30    每 30 秒发一次心跳
ServerAliveCountMax=10    10 次无回应才断开
ExitOnForwardFailure=yes  端口绑定失败时退出
```

云服务器端保活（`sshd_config`）：

```text
ClientAliveInterval 30
ClientAliveCountMax 10
```

## 腾讯云安全组

入站规则：

```text
TCP 22    - SSH
TCP 8000  - LiteLLM API Gateway
TCP 3000  - OpenWebUI（仅需要时开放，优先迁移到本地节点）
```

安全建议：

- API Key 不写入公开文档。
- 如果 8000 暴露公网，优先限制来源 IP。
- 后续引入 Nginx 或 Caddy 做 HTTPS、访问日志和限流。

## 已知问题

### NAT 回环

内网设备可能无法通过云服务器公网 IP 访问自己。

解决：5090 本机直接连接 `http://127.0.0.1:1234/v1`，外部客户端连接云服务器公网 Base URL。

### 服务器内存限制

2GB 内存不能稳定承载 OpenWebUI、RAG、Agent Runtime 等重服务。

当前策略：云服务器只做轻量入口；OpenWebUI、RAG、Agent Runtime、向量库等迁移到本地节点。
